\begin{problem}{Калькулятор}{\name.in}{\name.out}{}{}
В новой программе OpenCalculator появилась новая возможность - можно настроить, какие кнопки отображаются, а какие - нет. 
Если кнопка не отображается на экране, то ввести соответствующую цифру с клавиатуры или копированием из другой программы нельзя. 
Петя настроил калькулятор так, что он отображает только кнопки с цифрами $x$, $y$, $z$. 
Напишите программу, определяющую, сможет ли Петя ввести число $N$, а если нет, то какое минимальное количество кнопок надо дополнительно отобразить на экране для его ввода.

\InputFile
Сначала вводятся три различных числа из диапазона от 0 до 9: $x$, $y$ и $z$ (числа разделяются пробелами).
Далее вводится целое неотрицательное число $N$, которое Петя хочет ввести в калькулятор ($N \le 10000$).

\OutputFile
Выведите, какое минимальное количество кнопок должно быть добавлено для того, чтобы можно было ввести число $N$ (если число может быть введено с помощью уже имеющихся кнопок, выведите 0)
