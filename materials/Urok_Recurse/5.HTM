Рекурсивные рисунки и программы

 

Интересным является применение рекурсии при создании рисунков. Рассмотрим простейший рисунок  из окружностей разного радиуса (рис. 1).

 

Если вглядеться в рисунок внимательно, то можно заметить, что рисунок начинается с центральной окружности самого большого радиуса. Затем осуществляется переход на концы горизонтального диаметра окружности, которые должны играть роль центров двух окружностей меньшего радиуса (примерно в полтора раза). Этот же процесс повторяется и с этими двумя окружностями, и с полученными четырьмя новыми, и так далее до тех пор, пока уменьшающийся радиус окружности не станет меньше первоначального в 1,5×1,5×1,5×1,5 раз (если посчитать, то должно выполниться четыре вложенных вызова рекурсивной процедуры).

Рекурсивная процедура, выполняющая такой рисунок, должна иметь в качестве передаваемых параметров координаты центра окружности (X, Y) и величину радиуса (R).

 

Программа 1  (QBasic):

SUB RIS (X, Y, R)

     IF R < 11 THEN EXIT SUB              ‘выход из процедуры при радиусе R < 11

     CIRCLE (X, Y), R                        ‘рисуем окружность

     CALL RIS(X + R, Y, R / 1.5)                    ‘вызов рекурсивной процедуры для

     CALL RIS(X - R, Y, R / 1.5)              ‘рисования окружностей справа и слева

END SUB

CALL RIS(X, Y, R)                                         ‘первый вызов рекурсивной процедуры

Теперь, когда рекурсивный алгоритм создания рисунка из окружностей и линий стал понятен, можно рассмотреть более сложные, но в то же время и более красивые рисунки. Например, рисунки «Веточка симметричная» и «Веточка естественная», «Снежинки». При создании таких рисунков «из реальной жизни» можно проявить настоящее творчество:

-        снежинки различные по форме или падающие по всему экрану;

-        веточки разного цвета, разной пушистости и разного размера (в длину ветки, угол наклона и цвет вносится случайная составляющая;

-        а если на последнем уровне вложенности веточки рисовать желтые кружочки, то из голой зимней ветки можно получить пушистую весеннюю мимозу.

 


Задача 3. Написать программу, создающую рисунок 3 «Веточка симметричная».

 

 

 

Программа 3.1 (QBasic):

 

SCREEN 12                                                       ‘Графический режим

RANDOMIZE TIMER                                       ‘Датчик случайных чисел

WINDOW (0, 0)-(640, 480)                              ‘Декартова система координат

x = 100: y = 40: l = 160                                       ‘Задание начальных значений параметров

a = INT(RND * 40 + 40)

a = a * 3.14159 / 180

CALL ris(x, y, l, a)                           ‘Рекурсивное рисование ветки, первый вызов

 

SUB ris (x, y, l, a)                             ‘Параметры: x,y – координаты начала ветки, l – длина, a - угол

     IF l < 1 THEN EXIT SUB         ‘Выход из рекурсии при длине ветки меньше 1

     cc = 6                                        'Цвет

     IF l < 15 THEN cc = 14            'Цвет меняется при другой длине

     c = INT(RND * 25 + 15)

     c = c * 3.14159 / 180               ‘Перевод в радианы

     c1 = c + a                                 ‘Новый угол

     x1 = x + l * COS(a)                  ‘Координаты конца ветки

     1 = y + l * SIN(a)

     LINE (x, y)-(x1, y1), cc           ‘Рисуем ветку

     CALL ris(x1, y1, l * .65, c1)    ‘Рисуем левую подветку

     c = INT(RND * 15 + 25)         ‘Добавка к углу

     c = c * 3.14159 / 180               ‘Перевод в радианы

     c2 = a – c                                  ‘Новый угол

     CALL ris(x1, y1, l * .65, c2)      ‘Рисуем правую подветку

END SUB

 

Программа 3.2 (Pascal):

USES GRAPH;

VAR

    GD,GM,I,J:INTEGER;

PROCEDURE VETKA(X,Y,D:INTEGER;U:REAL);

VAR

    X1,Y1,D1:INTEGER;

    U1:REAL;

BEGIN

    IF D<5 THEN EXIT;

    IF D<8 THEN SETCOLOR(14)

     ELSE IF D<16 THEN SETCOLOR(2)

               ELSE SETCOLOR(6);

    X1:=X+ROUND(D*SIN(U));

    Y1:=Y+ROUND(D*COS(U));

    LINE(X,Y,X1,Y1);

    D1:={ROUND(D/1.3)}D-ROUND(RANDOM(D)/1.7);

    U1:=0.1+RANDOM*1;

    VETKA(X1,Y1,D1,U+U1);

    VETKA(X1,Y1,D1,U-U1);

END;

BEGIN

    RANDOMIZE;

    GD:=DETECT;

    INITGRAPH(GD,GM,'');

    IF GRAPHRESULT<>GROK THEN HALT(1);

    FOR I:=1 TO 30 DO BEGIN

        VETKA(120,340,60,2.5);

        READLN;

        CLEARDEVICE;

    END;

    CLOSEGRAPH;

END.

 

 

Задача 4. Написать программу, создающую рисунок 4 «Веточка естественная».

 

Указание: Рисунок представляет собой несимметричную веточку, по своему виду более похожую на настоящую «живую» ветку. Принцип построения этой веточки отличается от принципа построения предыдущей симметричной только тем, что в длину отрезков и угол наклона вносится случайная составляющая.

Рекурсивная процедура также имеет 4 параметра: координаты начала отрезка, его длину и угол наклона к оси ox.

 

Алгоритм построения состоит из следующих шагов:

1.               Вычислить координаты конца отрезка (известны координаты начала отрезка, его длина и угол наклона).

2.               Нарисовать отрезок.

3.               Вызвать рекурсивную процедуру, рисующую левую ветку, с новыми параметрами: за координаты начала отрезка взять только что вычисленные координаты конца отрезка, за длину – длину, уменьшенную в каких-то заданных пределах (может быть, от 1.5 до 3 раз),  за угол – первоначальный угол, уменьшенный на какое-то случайное число (например, от 5 до 60 градусов).

4.               Вызвать рекурсивную процедуру, рисующую правую ветку с параметрами такими  же, как в пункте 3, за исключением угла – новый угол увеличивается на случайное число (например, от 5 до 60 градусов).

5.               Выход из процедуры осуществляется, когда длина ветки становится достаточно малой (около 5 пикселей).

В основной программе имеется лишь одно обращение к процедуре со следующими параметрами: координаты начала самого первого отрезка, первоначальная длина отрезка и угол наклона.

SCREEN 12

x1 = 320: y1 = 240: r = 100

 SUB ris (x1, y1, r)

IF r < 5 THEN EXIT SUB

c = INT(RND * 15 + 1)

LINE (x1 - r, y1)-(x1 + r, y1), c

LINE (x1, y1 + r)-(x1, y1 - r), c

LINE (x1 - r / 2, y1 - r / 2)-(x1 + r / 2, y1 + r / 2), c

LINE (x1 + r / 2, y1 - r / 2)-(x1 - r / 2, y1 + r / 2), c

CALL ris(x1 - r, y1, r / 2)

CALL ris(x1 + r, y1, r / 2)

CALL ris(x1, y1 + r, r / 2)

CALL ris(x1, y1 - r, r / 2)

END SUB

 

 

Задача 6. Написать программу, создающую рисунок 6 «Снежинки десятиконечной».

 

Указание. Рисунок представляет собой десятиконечную снежинку, каждый конец которой, в свою очередь, представляет собой множество подобных десятиконечных снежинок, уменьшающихся в размере по направлению к центру.

 

Рекурсивная процедура, рисующая такую снежинку, зависит от трех параметров (трех чисел, однозначно характеризующих изображаемую фигуру): координат центра x, y и радиуса r.

Снежинка с радиусом r £ 1 на экране монитора не будет отличаться от пикселя, отсюда следует условие выхода из рекурсивной процедуры: если r £ 1, то рисуется лишь одна точка с координатами (x, y), и процедура завершается.

В противном случае (r > 1) в процедуре изображаются десять ветвей снежинки в цикле по углу с шагом 36°, начиная с крайней удаленной снежинки, радиусом в 5 раз меньшим начального радиуса (размера).

Внутри этого цикла рисуется собственно сама ветвь (от края к центру), которая состоит опять же из восьми подобных снежинок, причем радиус очередной снежинки уменьшается по мере приближения к центру исходной снежинки в 1.5 раза.

Для того чтобы нарисовать всю снежинку целиком, из основной программы достаточно вызвать эту процедуру один раз, передав в качестве параметров координаты центра экрана и радиус исходной снежинки.
