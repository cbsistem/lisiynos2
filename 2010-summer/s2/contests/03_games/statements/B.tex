\begin{problem}{Конфетки}{candy.in}{candy.out}{1 секунда}{64 мегабайта}

В кучке лежит $N$ конфеток. Двое по очереди делают ходы. На каждом ходе игрок может
взять от 1 до $K$ конфеток, но не больше, чем взял его противник на предыдущем ходе (при
первом ходе первого этого ограничения нет). Проигрывает тот, кто на своем ходе не сможет
взять конфетку. Сможет ли первый игрок выиграть, и как для этого он должен действовать?
(Считается, что второй игрок всегда делает оптимальные ходы).

\InputFile

Во входном файле записаны через пробел два числа ~--- $N$ ($1 \le N \le 500$) и $K$
($1 \le K \le 100$).
\OutputFile

В выходной файл выведите число конфет, которое должен взять первый игрок, чтобы
выиграть, либо 0, если первый игрок выиграть не может.

\Example

\begin{example}
\exmp{
7 3
}{
3
}%
\end{example}
Если в свой ход второй игрок берёт 2 или 3 конфеты, то следующим ходом первый игрок
заканчивает игру, если же второй игрок возьмёт одну конфету, то далее каждый будет брать
по одной конфете и последняя достанется первому игроку.

\end{problem}
